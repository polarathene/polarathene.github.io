Javascript and the DOM

An analogy to describe JavaScript and its relationship to HTML and CSS

The content of an HTML file describes the structure of a document. It provides contextual meaning to the text it contains, possibly with some images and links to reference other relevant documents.

That contextual meaning for text and other content allows software like a web browser to better understand and present the content to you the reader. But HTML does not communicate how to display larger text for headings, if the text should use columns, or how that text interacts with imagery, nor where that imagery should be located on the screen. This is the role of CSS, it provides control over how the website will look.

A good way to visualize this collaboration is a newspaper:

Before laying out all that content for a page, you would have it in a simpler form first. The image, different articles with paragraphs and their headings, references to other pages (your links).

CSS is what has the text flow in columns and in the bottom right, wrap around the image. The HTML content is unaware of this and doesn't need to be split up when a pragraph splits at the end of one column in our newspaper and starts at the top of a new column.

Today we find the news on the web or our phones, but HTML and CSS isn't quite enough. We interact with the content, controlling playback of a video, expressing an opinion through a comment section or hitting up a search field that responds with suggestions as we type. Javascript fulfills this role, providing or enhancing the functionality to serve these experiences.

Control flow using an example from everyday life

Control flow in programming is how we navigate through the code as it is processing, what path it takes. Sort of like how you would read through a newspapers columns and find a reference to read more on another page that you flip to before you finish reading the current page.

Initially, the control flow is pretty simple. We start at the top of the file and we work our way down line by line.

There are some control structures that can interrupt that flow, such as conditionals, functions, and loops.

  • Conditional statements as the name implies will only allow some code to run when an expectation is met.
  • Functions we will cover later in the article, but are similar to our page jumping with the newspaper, they perform an action somewhere else before continuing forward.
  • Loops define a section of code that can be repeated multiple times, which is suitable for condensing code exhibiting a sequential pattern.

A bus traveling along it's daily route is a good example of control flow playing out in the real world:

  1. Drive along the route.
  2. If a passenger signals they want to leave, drop them off at the next bus stop.
  3. If approaching a bus stop, and someone signals the bus driver, pick them up.
  4. Else continue driving. Repeat from step 1 until route is completed.

The difference between accessing data from arrays and objects

          
Arrays provide sequential sets of data that we can access by querying an index. Like a number in a queue or waiting list. We do this with the variable name followed by brackets:

          
pages[3] would be the fourth item in the pages array. It is the fourth because the index starts incrementing from 0, rather than 1.

          
Objects are similar to arrays, but rather than storing data as a list with an index, a value of data is associated to a "key", also known as a "property" of the object.

          
Typically "dot notation" syntax is used when referring to an explicit object key, such as book.pages, but you can also use "bracket notation" syntax instead with the key as a string; book["pages"] - that is more useful when the provided key is better suited as a variable, such as when used in loops or functions.

          
Objects like arrays can also be nested, and both can contain each other too.

What functions are and why they are helpful

          
When we write code and a section of it is to perform a specific task, that code can be grouped together into what is known as a function.

          
This is especially useful if we would have previously copy and pasted that the same code to perform that task again somewhere in our program. We have one common place to share the code when it is a function, and using it elsewhere is now only a single line! How awesome is that?

          
Beyond recycling our code and making it easier on ourselves to get a summary of what is going on; functions can also accomodate small changes with parameters. We can use these like variables, they are an input into the function, and our function can provide an output too if that'd be useful for what we're doing. Oh, and you can put functions within your functions of course!
